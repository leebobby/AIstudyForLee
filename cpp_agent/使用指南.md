## 1. 环境准备

需要安装以下工具（只需做一次）：

### 1.1 CMake（构建系统）

```bash
# Windows（推荐，自动配置环境变量）
winget install Kitware.CMake

# 验证安装
cmake --version
# 预期输出：cmake version 3.x.x
```

### 1.2 C++ 编译器（二选一）

**方案 A：MinGW-w64（轻量，推荐新手）**

```bash
winget install mingw
# 安装完重启终端

# 验证
g++ --version
# 预期输出：g++ (x86_64-posix-seh ...) 13.x.x
```

**方案 B：Visual Studio（已安装 VS 则无需额外操作）**

打开 Visual Studio Installer，确认勾选「使用 C++ 的桌面开发」工作负载即可。

### 1.3 Git（已安装，可跳过）

```bash
git --version
# 预期输出：git version 2.x.x
```

### 1.4 Python 3.9+

```bash
python --version
# 预期输出：Python 3.9.x 或更高
```

## 3. 配置 API Key

### 获取通义千问 API Key

1. 打开 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 进入「API-KEY 管理」→ 创建 API Key
4. 复制 Key（格式类似 `sk-xxxxxxxxxxxxxxxx`）

### 设置环境变量（推荐，Key 不写死在代码里）

```bash
# Windows CMD
set QWEN_API_KEY=sk-你的key

# Windows PowerShell
$env:QWEN_API_KEY="sk-你的key"

# 永久生效（写入系统环境变量）
setx QWEN_API_KEY "sk-你的key"
```

## 4. 配置 config.py

打开 [config.py](config.py)，根据需要修改以下参数：

```python
# ── 千问 API ────────────────────────────────────────────
QWEN_API_KEY  = os.getenv("QWEN_API_KEY", "your-api-key-here")
#               ↑ 优先读环境变量；或直接把 Key 填在这里（不推荐提交到 git）

# ── 工作目录 ─────────────────────────────────────────────
WORKSPACE_DIR = BASE_DIR / "workspace"
#               ↑ 生成的 C++ 项目放这里，默认是 cpp_agent/workspace/

# ── Git 设置 ─────────────────────────────────────────────
GIT_AUTO_PUSH = False   # True = 每次完成后自动 git push；False = 只 commit
GIT_BRANCH    = "main"  # push 到哪个分支

# ── Agent 行为 ───────────────────────────────────────────
MAX_FIX_RETRIES = 3     # 编译/测试失败后最多重试几次
```

> **常见配置场景：**
> - 只想本地用：`GIT_AUTO_PUSH = False`（默认）
> - 想自动推送到远程：`GIT_AUTO_PUSH = True`，并在 workspace 目录配置好 remote

## 8. 常见问题

### Q1：提示 `cmake: command not found`

**原因**：CMake 未安装或未加入 PATH。

**解决**：

```bash
winget install Kitware.CMake
# 安装完后重启终端（让 PATH 生效）
cmake --version  # 验证
```

---

### Q2：提示 `g++: command not found` 或编译报错找不到编译器

**原因**：没有 C++ 编译器。

**解决（MinGW）**：

```bash
winget install mingw
# 重启终端后验证
g++ --version
```

**解决（MSVC）**：打开 Visual Studio Installer → 修改 → 勾选「使用 C++ 的桌面开发」。

---

### Q3：提示 `QWEN_API_KEY` 未设置 或 API 返回 401

**原因**：API Key 未配置或已过期。

**解决**：

```bash
# 检查是否设置
echo %QWEN_API_KEY%

# 重新设置
set QWEN_API_KEY=sk-你的正确key
```

---

### Q4：编译失败且重试 3 次后仍失败

**原因**：需求过于复杂，或涉及本机没有的库（如 Boost）。

**解决**：
1. 查看 `workspace/<功能名>/src/` 下的生成代码，手动修改
2. 尝试更明确的需求描述（避免隐含的第三方库依赖）
3. 提高 `MAX_FIX_RETRIES = 5`（config.py）

---

### Q5：Agent 问了我问题，我怎么回答？

当需求模糊时，Agent 会提一个问题，例如：

```
[Analyzer] 需求有些模糊，请回答一个问题：
           消息队列容量是否有上限？有上限时生产者应该阻塞还是丢弃消息？
你的回答 > 有上限，生产者阻塞等待
```

直接输入你的补充说明并回车即可，Agent 会继续执行。

---

### Q6：想让 Agent 自动 push 到我的 GitHub

1. 先在 `workspace/` 目录添加 remote：

```bash
cd cpp_agent/workspace
git remote add origin https://github.com/你的用户名/你的仓库.git
```

2. 修改 `config.py`：

```python
GIT_AUTO_PUSH = True
GIT_BRANCH    = "main"
```

## 10. 与现有 C++ 项目集成

如果你已有 C++ 项目仓库，希望 Agent 直接把代码生成到该仓库：

### 步骤 1：修改 config.py

```python
# 把 WORKSPACE_DIR 指向你的项目根目录
WORKSPACE_DIR = Path(r"D:\MyProjects\my-cpp-lib")

# 如果仓库已有 remote，开启自动 push
GIT_AUTO_PUSH   = True
GIT_REMOTE_NAME = "origin"
GIT_BRANCH      = "main"
```

### 步骤 2：确认目录结构兼容

Agent 会在 `WORKSPACE_DIR/<feature_name>/` 下创建文件，不会修改其他目录。
你的项目结构示例：

```
my-cpp-lib/
├── existing_module/      ← 已有模块，Agent 不会动
├── lru_cache/            ← Agent 新建
│   ├── CMakeLists.txt
│   ├── src/
│   └── tests/
└── CMakeLists.txt        ← 已有的顶层 CMake（需手动 add_subdirectory）
```

### 步骤 3：把生成模块加入顶层 CMake（可选）

Agent 生成的每个功能都有独立的 `CMakeLists.txt`，可以在顶层 `CMakeLists.txt` 中引入：

```cmake
# 在你的顶层 CMakeLists.txt 末尾加上：
add_subdirectory(lru_cache)
add_subdirectory(thread_safe_queue)
```

## 11. 各脚本文件说明

```
cpp_agent/
│
├── config.py                  ← ⭐ 唯一需要你修改的配置文件
│
├── requirements.txt           ← Python 依赖，pip install 用
│
├── main.py                    ← 启动入口，交互循环在这里
│
└── agent/
    │
    ├── requirement_analyzer.py
    │   作用：把你的自然语言描述解析成结构化 Spec（JSON）
    │   调用：qwen-max
    │   关键输出：feature_name / public_api / edge_cases / confidence
    │
    ├── code_generator.py
    │   作用：根据 Spec 生成 .h / .cpp / CMakeLists.txt
    │   调用：qwen-coder-plus（专用代码模型，比 max 更擅长写代码）
    │   关键方法：generate() / fix_code()（编译失败时调用）
    │
    ├── build_system.py
    │   作用：调用 cmake 配置 + 编译，捕获错误信息
    │   不调用 AI，纯本地操作
    │   关键方法：configure_and_build() / build_tests()
    │
    ├── test_runner.py
    │   作用：生成测试文件 → 编译 → 运行 → 解析结果
    │   调用：qwen-coder-plus
    │   生成的测试：纯标准C++，不依赖 Google Test 等第三方框架
    │   关键方法：generate_tests() / run() / fix_source()
    │
    ├── git_manager.py
    │   作用：git add + 生成 commit message + commit（可选 push）
    │   调用：qwen-max（生成 Conventional Commits 格式的 message）
    │   关键方法：commit()
    │
    └── orchestrator.py
        作用：把上面 5 个模块串联起来，管理重试逻辑
        不直接调用 AI，只负责流程控制
        关键方法：run(requirement) → AgentResult
```

## 12. 调整 Agent 行为

### 12.1 更换模型

打开 [config.py](config.py) 修改：

```python
MODEL_CHAT  = "qwen-max"        # 需求理解、commit message 用
MODEL_CODER = "qwen-coder-plus" # 代码生成、测试生成、修复 用

# 可选值（按能力从高到低）：
# qwen-max            → 最强推理，适合复杂需求分析
# qwen-coder-plus     → 代码专用，生成质量最佳（推荐）
# qwen-plus           → 性价比高，适合简单需求
# qwen-turbo          → 最快最便宜，适合测试阶段
```

### 12.2 增加重试次数（需求复杂时）

```python
MAX_FIX_RETRIES = 5   # 默认 3，复杂项目可以调高
```

### 12.3 调低澄清门槛（让 Agent 更多提问）

```python
CLARIFY_THRESHOLD = 0.8   # 默认 0.6；越高 = Agent 越容易追问
#                           0.0 = 从不提问，直接按自己理解做
#                           0.9 = 几乎每次都会提问确认
```

### 12.4 修改 Prompt（定制 Agent 风格）

各模块的 `_SYSTEM_PROMPT` 变量在对应文件顶部，可以直接编辑：

| 想修改的行为 | 编辑的文件 | 变量名 |
|---|---|---|
| 需求理解方式 | `agent/requirement_analyzer.py` | `_SYSTEM_PROMPT` |
| 代码风格（注释/命名） | `agent/code_generator.py` | `_SYSTEM_PROMPT` |
| 测试覆盖侧重 | `agent/test_runner.py` | `_SYSTEM_PROMPT` |

**例：让生成的代码强制加中文注释**

打开 [agent/code_generator.py](agent/code_generator.py)，在 `_SYSTEM_PROMPT` 中加一行：

```python
_SYSTEM_PROMPT = """\
...
- 关键函数和类必须有中文注释说明用途
...
"""
```
