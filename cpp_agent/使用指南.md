## 1. 环境准备

需要安装以下工具（只需做一次）：

### 1.1 CMake（构建系统）

```bash
# Windows（推荐，自动配置环境变量）
winget install Kitware.CMake

# 验证安装
cmake --version
# 预期输出：cmake version 3.x.x
```

### 1.2 C++ 编译器（二选一）

**方案 A：MinGW-w64（轻量，推荐新手）**

```bash
winget install mingw
# 安装完重启终端

# 验证
g++ --version
# 预期输出：g++ (x86_64-posix-seh ...) 13.x.x
```

**方案 B：Visual Studio（已安装 VS 则无需额外操作）**

打开 Visual Studio Installer，确认勾选「使用 C++ 的桌面开发」工作负载即可。

### 1.3 Git（已安装，可跳过）

```bash
git --version
# 预期输出：git version 2.x.x
```

### 1.4 Python 3.9+

```bash
python --version
# 预期输出：Python 3.9.x 或更高
```

## 3. 配置 API Key

### 获取通义千问 API Key

1. 打开 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 进入「API-KEY 管理」→ 创建 API Key
4. 复制 Key（格式类似 `sk-xxxxxxxxxxxxxxxx`）

### 设置环境变量（推荐，Key 不写死在代码里）

```bash
# Windows CMD
set QWEN_API_KEY=sk-你的key

# Windows PowerShell
$env:QWEN_API_KEY="sk-你的key"

# 永久生效（写入系统环境变量）
setx QWEN_API_KEY "sk-你的key"
```

## 4. 配置 config.py

打开 [config.py](config.py)，根据需要修改以下参数：

```python
# ── 千问 API ────────────────────────────────────────────
QWEN_API_KEY  = os.getenv("QWEN_API_KEY", "your-api-key-here")
#               ↑ 优先读环境变量；或直接把 Key 填在这里（不推荐提交到 git）

# ── 工作目录 ─────────────────────────────────────────────
WORKSPACE_DIR = BASE_DIR / "workspace"
#               ↑ 生成的 C++ 项目放这里，默认是 cpp_agent/workspace/

# ── Git 设置 ─────────────────────────────────────────────
GIT_AUTO_PUSH = False   # True = 每次完成后自动 git push；False = 只 commit
GIT_BRANCH    = "main"  # push 到哪个分支

# ── Agent 行为 ───────────────────────────────────────────
MAX_FIX_RETRIES = 3     # 编译/测试失败后最多重试几次
```

> **常见配置场景：**
> - 只想本地用：`GIT_AUTO_PUSH = False`（默认）
> - 想自动推送到远程：`GIT_AUTO_PUSH = True`，并在 workspace 目录配置好 remote

## 5. 运行 Agent

### 方式 A：交互模式（推荐）

```bash
cd cpp_agent
python main.py
```

启动后会显示提示符，直接输入需求：

```
请输入需求 > 实现一个线程安全的消息队列
```

### 方式 B：直接传参（适合脚本调用）

```bash

```

### 退出交互模式

输入 `quit` 或 `q` 或按 `Ctrl+C`。

## 8. 常见问题

### Q1：提示 `cmake: command not found`

**原因**：CMake 未安装或未加入 PATH。

**解决**：

```bash
winget install Kitware.CMake
# 安装完后重启终端（让 PATH 生效）
cmake --version  # 验证
```

---

### Q2：提示 `g++: command not found` 或编译报错找不到编译器

**原因**：没有 C++ 编译器。

**解决（MinGW）**：

```bash
winget install mingw
# 重启终端后验证
g++ --version
```

**解决（MSVC）**：打开 Visual Studio Installer → 修改 → 勾选「使用 C++ 的桌面开发」。

---

### Q3：提示 `QWEN_API_KEY` 未设置 或 API 返回 401

**原因**：API Key 未配置或已过期。

**解决**：

```bash
# 检查是否设置
echo %QWEN_API_KEY%

# 重新设置
set QWEN_API_KEY=sk-你的正确key
```

---

### Q4：编译失败且重试 3 次后仍失败

**原因**：需求过于复杂，或涉及本机没有的库（如 Boost）。

**解决**：
1. 查看 `workspace/<功能名>/src/` 下的生成代码，手动修改
2. 尝试更明确的需求描述（避免隐含的第三方库依赖）
3. 提高 `MAX_FIX_RETRIES = 5`（config.py）

---

### Q5：Agent 问了我问题，我怎么回答？

当需求模糊时，Agent 会提一个问题，例如：

```
[Analyzer] 需求有些模糊，请回答一个问题：
           消息队列容量是否有上限？有上限时生产者应该阻塞还是丢弃消息？
你的回答 > 有上限，生产者阻塞等待
```

直接输入你的补充说明并回车即可，Agent 会继续执行。

---

### Q6：想让 Agent 自动 push 到我的 GitHub

1. 先在 `workspace/` 目录添加 remote：

```bash
cd cpp_agent/workspace
git remote add origin https://github.com/你的用户名/你的仓库.git
```

2. 修改 `config.py`：

```python
GIT_AUTO_PUSH = True
GIT_BRANCH    = "main"
```
