## 一、业务诊断 / 硬件诊断 — 运行脚本

**场景：快速诊断一台节点的检测流程是否正常**

**第 1 步：切换到对应 Tab**

打开左侧菜单「故障诊断」→ 点击顶部 **「业务诊断」** 标签页。

**第 2 步：找到脚本并运行**

在「检测流程」分类下找到目标脚本卡片（如「数据流检测」），点击卡片底部的 **蓝色「▶ 运行」按钮**。

**第 3 步：填写运行参数**

| 字段 | 填写示例 | 说明 |
|------|----------|------|
| 目标节点 | master-1 | 从下拉列表多选，节点来自系统节点表 |
| 用户名 | root | OpenEuler SSH 登录用户 |
| 密码 | xxxxxx | SSH 登录密码 |
| 端口 | 22 | 默认 22，有改过再修改 |

**第 4 步：点击「▶ 执行」**

对话框下方会出现每个节点的结果块：

- 绿色 `exit 0` — 执行成功，stdout 区域显示命令输出
- 红色 `exit 1` — 命令本身执行失败，stderr 区域显示错误原因
- 红色 `exit -2` — 执行超时，被系统强制中断

## 二、新建诊断类型

**场景：公司新增了「压测流程」业务，需要一个独立分类**

1. 在「业务诊断」Tab 点击工具栏的 **「📁 新建诊断类型」**
2. 弹窗中输入类型名称（如 `压测流程`），按 **Enter** 或点「创建」
3. 页面新增「压测流程」区块（带红色分隔线），等待添加脚本

> **注意：** 自定义类型保存在浏览器 localStorage，刷新页面不丢失；清除浏览器数据后需重新添加，建议配合「导出配置」做备份。

## 三、新建脚本

**场景：在「压测流程」下新增一个检查 CPU 压力的脚本**

**入口（二选一）：**

- 工具栏点 **「+ 新建脚本」**（手动选分类）
- 空分类卡片中点 **「+ 添加脚本」**（自动预填分类）

**填写表单：**

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 脚本名称 | CPU 压力检测 | 简洁，显示在卡片标题 |
| 所属分类 | 压测流程 | 输入时有 autocomplete 提示已有分类 |
| 节点类型 | all | 限定脚本适用的节点类型（all / master / slave / sensor） |
| 描述 | 运行 stress-ng 查看 CPU 响应 | 显示在卡片副标题（可选） |
| 脚本内容 | 见下方示例 | SSH 到节点后执行的 shell 命令 |
| 超时(秒) | 60 | 超时后强制中断，返回 exit -2 |
| 启用 | 开 | 关闭后卡片半透明，「运行」按钮置灰 |

**脚本内容示例：**

```bash
echo "=== CPU 核心数 ==="
nproc
echo "=== 5s 负载测试 ==="
stress-ng --cpu 4 --timeout 5s --metrics 2>&1 | tail -10
```

**导入本地文件（可选）：** 若脚本已写好在 `.sh` 文件里，点「📤 导入本地脚本文件」选择文件，内容自动填入编辑框。

点 **「保存」** 后，脚本卡片立即出现在对应分类下。

## 四、修改已有脚本

**场景：把「内核错误日志」的搜索范围从最近 100 行扩大到 500 行**

1. 在「硬件诊断 → 系统诊断」找到「内核错误日志」卡片
2. 点卡片底部的 **「✏ 编辑」按钮**（铅笔图标）
3. 在脚本内容框中修改命令：

```bash
# 修改前
dmesg | grep -iE 'error|warn|fail|oops|panic' | tail -100

# 修改后
dmesg | grep -iE 'error|warn|fail|oops|panic' | tail -500
```

4. 点 **「保存」**，修改立即生效（下次运行使用新内容）

**可修改的所有字段：**

| 字段 | 修改说明 |
|------|----------|
| 脚本名称 | 直接改，卡片标题同步更新 |
| 所属分类 | 改分类后卡片会移动到新分类下 |
| 节点类型 | 影响「运行」弹窗中的节点下拉范围 |
| 描述 | 修改卡片副标题 |
| 脚本内容 | 可直接编辑，也可重新导入文件覆盖 |
| 超时(秒) | 按实际命令执行时长设置，长时脚本适当增大 |
| 启用开关 | 关闭=临时禁用但不删除 |

## 五、删除脚本

点卡片底部的 **「🗑 删除」按钮**（红色）→ 确认弹窗中点「删除」。

> 删除前弹窗会显示脚本名称作为二次确认，操作不可逆，建议删除前先「导出配置」备份。

## 六、脚本配置的导出与导入

页面顶部「**脚本配置**」操作栏支持配置的跨环境迁移。

### 导出配置

点 **「↓ 导出配置」** → 浏览器下载 `diag_scripts.json`，包含所有脚本的完整定义。

### 导入配置（在新环境恢复）

点 **「↑ 导入配置」** → 选择之前导出的 `.json` 文件 → 系统按脚本名称**合并**：

- 同名脚本：更新内容
- 新名脚本：新增
- 不会产生重复脚本

### 保存为发布配置

点 **「📦 保存为发布配置」** → 将当前所有脚本固化为基线版本，顶部状态栏显示：

```
已有发布包 (N 脚本)
```

鼠标悬浮在按钮上可查看脚本数量和最后更新时间，适合在正式投产前锁定脚本集合。

## 七、日志导出 — 从 OpenEuler 节点拉取日志

**场景：某节点告警，需要收集近 30 分钟的业务日志保存到本地**

### 7.1 填写 SSH 连接信息

| 字段 | 示例 | 说明 |
|------|------|------|
| 目标主机 | 192.168.1.10 | 节点管理面 IP |
| SSH 端口 | 22 | 默认 22 |
| 用户 | root | SSH 登录用户 |
| 密码 | xxxxxx | 留空则使用上次保存的密码 |
| 输出目录 | `D:\logs\export` | Windows 本地路径，不存在自动创建 |

### 7.2 设置告警时间窗（可选）

若要按告警时间段过滤日志：

1. 填写「告警时间」，如 `2024-05-18 14:30:00`
2. 设置时间窗：前 `30` 分，后 `5` 分
3. 脚本中可使用三个环境变量：

| 变量 | 含义 | 示例值 |
|------|------|--------|
| `$ALERT_TIME` | 告警时刻 | `2024-05-18T14:30:00` |
| `$ALERT_FROM` | 窗口开始时间 | `2024-05-18T14:00:00` |
| `$ALERT_TO` | 窗口结束时间 | `2024-05-18T14:35:00` |

### 7.3 勾选采集脚本

「采集脚本」区域按类型分组，勾选方式：

- 单独勾选每张脚本卡片的复选框
- 点分类标题旁的 **「全选」** 复选框批量选中整个分类
- 再次点「全选」取消该分类的所有勾选

### 7.4 开始导出

点 **「↓ 开始导出 (N 个脚本)」** 按钮，下方结果表格实时滚动更新：

| 列 | 说明 |
|----|------|
| 类型 | 采集脚本所属分类 |
| 脚本 | 脚本名称 |
| 模式 | `stdout 落盘` 或 `SFTP 完整文件` |
| 状态 | 成功 / 失败（含退出码） |
| 大小 | 落盘文件大小（KB） |
| 远端路径 | SFTP 模式下的远端源文件路径 |
| 本地落盘路径 | 文件保存在本机的完整路径 |

**中途停止：** 点红色 **「⊗ 终止」** 按钮，当前正在执行的脚本被中断，已完成的文件保留。

## 八、脚本的两种工作模式

新建/编辑**日志导出**类脚本时需选择输出模式：

| 模式 | 标签颜色 | 工作方式 | 适用场景 |
|------|----------|----------|----------|
| **stdout 落盘** | 蓝色 | 执行命令，把 stdout 写成 `.log` 文件 | `dmesg`、`journalctl`、`ps aux` 等命令 |
| **SFTP 完整文件** | 绿色 | 脚本输出文件路径列表，系统逐行 SFTP 下载 | `/var/log/messages` 等大文件原样拷贝 |

**SFTP 模式脚本写法示例：**

```bash
# 每行输出一个远端文件的绝对路径
echo /var/log/messages
echo /var/log/secure
echo /var/log/audit/audit.log

# 也可以用 find 动态发现文件
find /var/log/app -name "*.log" -mmin -60

# 空行和 # 开头的注释行会被自动忽略
# echo /var/log/ignored.log
```

**结合告警时间窗的 stdout 模式示例：**

```bash
# 用 journalctl 按时间段导出系统日志
journalctl --since "$ALERT_FROM" --until "$ALERT_TO" --no-pager
```

## 九、常见问题

| 现象 | 原因 | 处理方式 |
|------|------|----------|
| 运行返回 `exit -2` | 命令执行超时被强制中断 | 编辑脚本，增大「超时(秒)」字段 |
| 运行返回 `exit -1`，stderr 显示"连接拒绝" | IP / 端口 / 密码错误，或防火墙拦截 | 检查节点管理面 IP 和 SSH 配置 |
| 导出文件大小为 0 KB | 远端文件为空，或权限不足无法读取 | 改用 stdout 模式，或脚本中加 `sudo` |
| 新建的自定义类型刷新后消失 | 浏览器 localStorage 被清除 | 重新添加；提前通过「导出配置」备份 |
| 脚本卡片显示为半透明 | 该脚本处于「禁用」状态 | 点「✏ 编辑」→ 打开「启用」开关 → 保存 |
| SFTP 模式提示"权限被拒绝" | 目标文件需要 root 权限才能读取 | 脚本改用 `sudo cat <path>` 输出，切换为 stdout 模式 |
| 导入配置后脚本数量没变化 | JSON 文件中的脚本名称与现有脚本同名，触发更新而非新增 | 正常行为；若需保留两份，改脚本名后再导入 |
