核心思想： 目标节点本身没有 OS，上电后 NIC 固件（PXE ROM）发出 DHCP 广播，从网络获取 IP 和引导程序，加载内核后由 Kickstart 文件驱动全自动安装——整个过程无需插 U 盘或光驱，数十台机器可同时并行装机。
1
BIOS/UEFI 选择网络启动 固件层

服务器上电，BIOS/UEFI 按启动顺序选中 Network Boot (PXE)。NIC 上的 PXE ROM（约 32KB 固件）被激活，它内置了一个极简 DHCP 客户端和 TFTP 客户端。此时机器没有 OS，完全靠这块固件运作。

2
DHCP 四步握手 — 获取 IP 和引导信息 DHCP / UDP
① DISCOVER
广播，我要IP
→
② OFFER
给你10.0.0.101
→
③ REQUEST
我要这个IP
→
④ ACK
确认+引导信息

DHCP ACK 中携带两个关键 Option：Option 66（next-server，TFTP 服务器 IP）和 Option 67（filename，引导文件路径如 pxelinux.0）。节点凭此知道去哪里下载什么。

3
TFTP 下载引导加载程序（Bootloader） TFTP / UDP 69

节点向 TFTP 服务器请求引导文件。TFTP 使用 UDP，无连接、无认证，适合固件级别的简单传输（但速度较慢，大文件后续改用 HTTP）。

下载顺序：pxelinux.0 → menu.c32（菜单模块）→ 读取 pxelinux.cfg/default（启动菜单配置）→ 节点显示启动菜单。

MAC 地址匹配：pxelinux 按优先级查找配置文件：先找 01-{MAC}，再找 IP 十六进制，最后 fallback 到 default。可为每台机器定制启动行为。

4
TFTP 下载内核和初始内存盘 TFTP / UDP 69

引导程序根据菜单配置，继续从 TFTP 下载两个核心文件：

• vmlinuz — 压缩的 Linux 内核，约 10-15 MB
• initrd.img — 初始内存文件系统，包含安装程序（Anaconda/d-i）及基础驱动，约 100-200 MB

内核启动参数由菜单中 APPEND 行指定，其中最关键的是 inst.ks=http://...（Kickstart 文件位置）和 inst.repo=http://...（安装镜像位置）。

5
内核启动 → 安装程序运行 → 读取 Kickstart HTTP / Kernel

内核解压 initrd，启动安装程序（Anaconda）。Anaconda 通过 HTTP 请求 Kickstart 文件，按其中的指令全自动完成：磁盘分区 → 格式化 → 通过 HTTP 拉取 RPM 包 → 安装 → 写入 bootloader → 配置网络/用户/SSH → 执行 %post 脚本。

全程无交互，100 台机器可并行执行，互不干扰。

6
安装完成 → 重启 → Ansible 接管 SSH / Ansible

节点安装完毕自动重启，这次从本地磁盘引导进入新系统。%post 脚本已预埋 SSH 公钥，Ansible 可立即 SSH 登录，推送集群配置：主机名、内核参数、容器运行时、Kubernetes 组件，最后执行 kubeadm init/join 组建集群。

🔷 TFTP 协议细节

端口69 / UDP
传输模式octet（二进制）
块大小默认 512B，可协商至 1468B
特点无认证、无加密、简单可靠
缺点速度慢，不适合大文件
传输内容pxelinux.0, vmlinuz, initrd
替代方案iPXE 可直接用 HTTP 传输
根目录/var/lib/tftpboot/

🟢 HTTP 协议角色

端口80 / TCP
用途1承载 OS 安装镜像 (repo)
用途2提供 Kickstart 文件
优势支持并发、速度快、断点续传

Kickstart（RedHat 系）/ Preseed（Debian 系）是安装自动应答文件。安装程序启动时读取这个文件，按其中的声明式指令全程自动完成安装，无需人工交互。一份 Kickstart 文件可驱动成百上千台机器安装出完全一致的系统。

🔑 核心要点总结

为什么用 TFTP 不用 HTTP 传内核？
PXE ROM 固件极其简单，只内置了 TFTP 客户端。HTTP 需要 TCP 栈和更复杂的实现。iPXE 是改进方案，支持 HTTP/HTTPS，可链式加载替换原生 PXE ROM。
如何区分每台机器的角色？
三种方式：① DHCP 按 MAC 绑定固定 IP，② pxelinux.cfg 按 MAC 文件名定制菜单，③ 统一装机后 Ansible 按 inventory 分组推送不同角色配置（推荐）。
大规模部署的瓶颈在哪？
① TFTP 速度慢 → 用 iPXE + HTTP 改善；② HTTP 服务器带宽 → 用 squid 缓存或部署多个镜像源；③ DHCP 广播风暴 → 独立部署 VLAN 隔离。
BIOS vs UEFI 的区别？
BIOS 用 pxelinux.0（syslinux），UEFI 用 grubx64.efi（grub2）。DHCP 通过 vendor-class-identifier（Option 60）区分，服务器端配置条件判断返回不同 filename。
安全注意事项
① 部署网络与生产网络 VLAN 隔离；② DHCP 仅允许已知 MAC 地址；③ Kickstart 密码使用哈希；④ 装机完成后关闭 PXE 启动顺序，防止误触发重装。
