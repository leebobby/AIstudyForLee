1

准备 PXE Host（手工，仅一次）

在 PXE Host 上安装 DHCP + TFTP + HTTP。构建单一 Base 黄金镜像（含所有角色需要的驱动：mlx5、ixgbe 等）。填写 nodes.json（22 台节点的 MAC→配置映射）。启动 node-config API 服务。

2

优先部署 SubSwath + Global Storage（NFS Server先就绪）

这4台机器提供 NFS 服务，Slave 节点 firstboot 时需要挂载 NFS，所以 必须先于 Slave 完成部署。PXE 启动 → 克隆镜像 → firstboot 安装 nfs-utils / opengauss，配置 /etc/exports，启动 nfs-server。完成后 NFS 服务在 172.16.x.x 上线。

3

部署 Master 节点（7台并行）

PXE 启动 → 克隆 base.img → firstboot 执行：配置 5 块网卡（1×10G + 2×100G DPDK + 2×100G RDMA）→ 设置大页内存 100×1G（写 grub，触发 reboot）→ reboot 后大页生效，DPDK/RDMA 口就绪。Master 间业务配置按业务层自行完成，部署层只负责 OS 和网络就绪。

4

部署 Slave 节点（12台并行）

PXE 启动 → 克隆 base.img → firstboot 执行：配置 10G 控制面口 + 100G RDMA 口 → 在 100G 口上添加辅助地址（172.16.x.x）用于 NFS 路由 → 挂载 NFS（subswath 和 gstorage）→ 写入 /etc/fstab 持久化。12台可完全并行，互不依赖。

5

验证全网连通性

从 PXE Host 运行连通性检查脚本：ping 所有 10G 控制面地址 → 从 Master ping DPDK/RDMA 地址 → 从 Slave 验证 NFS 挂载点可读写 → 从 Master ping Slave RDMA 地址。全部通过后集群网络层就绪，交由业务层继续初始化。

各角色 firstboot 中网络配置脚本的核心逻辑：
Master — 20-network.sh
# 读取 env
source /etc/node-role/env
IDX=${HOSTNAME_NEW##*-} # 提取编号，master-03 → 03

# 10G 控制面口
nmcli con add type ethernet ifname eth0 con-name ctrl \
  ip4 10.0.0.1${IDX}/24 gw4 10.0.0.1

# 100G DPDK 口（不配IP，交给 DPDK 接管）
nmcli con add type ethernet ifname eth1 con-name dpdk1 ipv4.method disabled
nmcli con add type ethernet ifname eth2 con-name dpdk2 ipv4.method disabled
# 绑定 dpdk 驱动
for nic in $DPDK_NICS; do
  echo "vfio-pci" > /sys/bus/pci/devices/$(ethtool -i $nic | awk '/bus-info/{print $2}')/driver_override
done

# 100G RDMA 口
nmcli con add type ethernet ifname eth3 con-name rdma1 \
  ip4 100.1.1.1${IDX}/24
nmcli con add type ethernet ifname eth4 con-name rdma2 \
  ip4 100.1.2.1${IDX}/24

nmcli con up ctrl && nmcli con up rdma1 && nmcli con up rdma2
Slave — 20-network.sh
source /etc/node-role/env
IDX=${HOSTNAME_NEW##*-}

# 10G 控制面
nmcli con add type ethernet ifname eth0 con-name ctrl \
  ip4 10.0.0.2${IDX}/24 gw4 10.0.0.1

# 100G RDMA 口，同时加辅助地址用于访问 NFS server（172.16网段）
nmcli con add type ethernet ifname eth1 con-name rdma \
  ip4 100.1.1.2${IDX}/24
# 添加到 172.16 网段的路由（通过同一块网卡）
ip route add 172.16.0.0/16 dev eth1

# NFS 挂载
mkdir -p /mnt/swath /mnt/global
for mount_entry in $NFS_MOUNTS; do
  SRC=$(echo $mount_entry | cut -d: -f1)
  DST=$(echo $mount_entry | cut -d: -f2)
  mount -t nfs -o rdma,port=20049 $SRC $DST
  echo "$SRC $DST nfs rdma,port=20049,_netdev 0 0" >> /etc/fstab
done
SubSwath — 20-network.sh + 80-nfs-server.sh
source /etc/node-role/env
IDX=${HOSTNAME_NEW##*-}

# 10G 控制面
nmcli con add type ethernet ifname eth0 con-name ctrl \
  ip4 10.0.0.4${IDX}/24 gw4 10.0.0.1

# 2×100G bond → NFS Server
nmcli con add type bond ifname bond0 con-name nfs-bond \
  bond.options "mode=active-backup,miimon=100" \
  ip4 172.16.1.4${IDX}/24
nmcli con add type ethernet ifname eth1 master bond0
nmcli con add type ethernet ifname eth2 master bond0
nmcli con up nfs-bond

# NFS Server 配置（80-nfs-server.sh）
dnf install -y nfs-utils opengauss
mkdir -p /data/export
echo "/data/export 100.1.1.0/24(rw,sync,no_root_squash)" >> /etc/exports
exportfs -ra
systemctl enable --now nfs-server
