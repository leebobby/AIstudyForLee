Step 1 — 准备构建环境（参考机 或 容器/chroot）

方式 A
参考机

在一台真实硬件上装好 OS，配置完所有东西，然后用 partclone 导出整盘镜像。最真实，驱动测试到位。

vs

方式 B
chroot 构建

在 PXE 服务器上 mount loop 基础 ISO，dnf/apt install 到一个 chroot 目录，最后 dd 成镜像。可脚本化重复构建，推荐用于 Base 层。

# 方式 B: chroot 构建 Base 镜像示例 BUILDROOT=/opt/buildroot mkdir -p $BUILDROOT # 挂载 OS ISO 作为包源 mount -o loop Rocky-9-latest-x86_64-dvd.iso /mnt/iso dnf --installroot=$BUILDROOT --releasever=9 \ --repo=file:///mnt/iso \ groupinstall "Minimal Install" -y # 挂载 proc/sys/dev（chroot 必须） mount --bind /proc $BUILDROOT/proc mount --bind /sys $BUILDROOT/sys mount --bind /dev $BUILDROOT/dev # 进入 chroot 执行配置 chroot $BUILDROOT /bin/bash -c " # 安装 100GE 网卡驱动（mlx5/ixgbe 等） rpm -ivh /tmp/drivers/mlx5-*.rpm modprobe mlx5_core echo 'mlx5_core' >> /etc/modules-load.d/mlx5.conf # 大页内存 sed -i 's/GRUB_CMDLINE_LINUX=\"/GRUB_CMDLINE_LINUX=\"default_hugepagesz=1G hugepagesz=1G hugepages=64 /' /etc/default/grub grub2-mkconfig -o /boot/grub2/grub.cfg # 内核调优参数 cat >> /etc/sysctl.d/99-base.conf << 'EOF' net.core.rmem_max = 134217728 net.core.wmem_max = 134217728 net.ipv4.ip_forward = 1 vm.swappiness = 0 EOF # SSH 公钥（管理节点） mkdir -p /root/.ssh && chmod 700 /root/.ssh echo 'ssh-rsa AAAA...YOUR_KEY...' > /root/.ssh/authorized_keys chmod 600 /root/.ssh/authorized_keys # 安装 firstboot service cp /tmp/firstboot.sh /usr/local/bin/firstboot.sh chmod +x /usr/local/bin/firstboot.sh cp /tmp/firstboot.service /etc/systemd/system/ systemctl enable firstboot "

Step 2 — 叠加角色配置（在 BUILDROOT 基础上 patch）

# 以"计算节点"为例，在 base buildroot 上叠加 apply_role() { local ROLE=$1 # compute / storage / database / mgmt cp -a $BUILDROOT ${BUILDROOT}-${ROLE} chroot ${BUILDROOT}-${ROLE} /bin/bash /tmp/roles/${ROLE}.sh } apply_role compute apply_role storage apply_role database apply_role mgmt

Step 3 — 打包为压缩镜像

# 方法 A: tar 打包（推荐，灵活，支持增量更新） tar -czf /srv/images/base-v1.0.tar.zst \ --zstd --numeric-owner -C $BUILDROOT . # 方法 B: partclone 块设备镜像（适合已装好的参考机） partclone.ext4 -c -s /dev/sda3 | zstd > /srv/images/compute-v1.0.img.zst # 校验文件 sha256sum /srv/images/*.zst > /srv/images/SHA256SUMS # 放到 HTTP 服务器 # 访问：http://10.0.0.1/images/compute-v1.0.img.zst

Step 4 — 构建极简部署 initrd

# 极简 initrd 只需要：busybox + wget/curl + zstd + partclone + lvm/mdadm # 约 30-50MB，用 dracut 定制或手工构建 dracut --force --no-hostonly \ --add "network base" \ --omit "plymouth biosdevname" \ --install "wget zstd partclone.ext4 sgdisk mkfs.ext4 mkfs.xfs" \ /var/lib/tftpboot/deploy-initrd.img \ $(uname -r)

关键设计原则： 四个镜像文件相互独立，互不依赖。Base 层的修改通过重新运行构建脚本同时更新四个镜像，角色脚本只维护各自的差异配置，保持 100-200 行以内。镜像版本号管理用 Git 跟踪构建脚本，不跟踪镜像文件本身（文件太大）。
firstboot.service 是一个 systemd oneshot 服务，镜像里预装，第一次启动时执行，执行完毕后自删并 disable 自身，永不再运行。它承担所有"机器级"的个性化动作，即镜像里无法预知的信息：IP、hostname、加入集群的 token 等。
1

探测自身身份

读取本机 MAC 地址，向 PXE 服务器查询：curl http://10.0.0.1/api/node-config?mac=aa:bb:cc:dd:ee:ff
服务器返回 JSON：hostname、static IP、gateway、role、cluster_token。PXE 服务器维护一张 MAC→配置 的静态表（简单 JSON 文件即可）。

2

设置 hostname 和静态 IP

hostnamectl set-hostname compute-01
用 nmcli 或直接写 /etc/NetworkManager/system-connections/ 配置静态 IP，然后 systemctl restart NetworkManager。

3

执行角色特定的加入动作

根据 role 字段分支执行：
compute/mgmt → kubeadm join ...
storage → ceph-volume lvm create ...
database → postgresql-setup --initdb

4

上报完成状态，自删

curl 通知 PXE 服务器该节点已就绪，然后：
systemctl disable firstboot && rm /etc/systemd/system/firstboot.service
此后节点正常运行，firstboot 永远不会再执行。

PXE 服务器的 node-config API（极简实现）
# /srv/node-config.json — MAC 到配置的静态映射表 { "aa:bb:cc:dd:ee:01": { "hostname": "compute-01", "ip": "10.1.0.101", "gateway": "10.1.0.1", "role": "compute", "cluster_token": "abc123.xyz..." }, "aa:bb:cc:dd:ee:02": { "hostname": "storage-01", "ip": "10.1.0.201", "role": "storage" } } # 用 Python 起一个极简 HTTP API，30 行搞定 # 或者直接用 nginx + lua，根据 ?mac= 参数返回对应 JSON 片段
